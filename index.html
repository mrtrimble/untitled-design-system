Accordions

Accordion Description

Accordion components are collapsible disclosure type elements often stacked into groups.

Accordions consist of a clickable header type element that toggles the visibility of the accordion's content section.

There are two types of accordions available, stylized <details> and <summary> HTML elements, as well as a JavaScript-enabled accordion using semantic markup.

Accordion Accessibility

  • Toggling of accordions should happen by clicking the header, or by pressing the Space or Enter keys while focused on the accordion.
  • The non-JavaScript variation of accordion follows the same semantics and accessibility of the <details> and <summary> HTML elements.
  • As for the JavaScript enhanced accordion, there are several things to consider:
    • The accordion header and accordion content elements must have unique IDs assigned to them.
    • Using a <div> element as the clickable header is not a good idea, but we can apply the role as button to let the browser pretend it is a button. (It's still not though!)
    • We need to apply two ARIA attributes to the header, aria-controls and aria-expanded.
    • aria-controls points the header at the unique ID of the accordion component.
    • aria-expanded keeps the element and assitive technology device up-to-date on the state of the accordion.
    • For the accordion content, include the role of region.
    • We will also need to specify that the content is labelled by the header by setting the aria-labelledby as the unique ID of the accordion header.

Accordion Progressive Enhancement

  • By including the no-js class on the body element, we can enhance the accordion with a gentle fallback.
    • By default the accordion components are set to an expanded state.
    • This ensures that if a user decides to browse with JavaScript disabled or the JavaScript fails to load, the content of the accordion will still be available.

Buttons

Button Description

There are three different types of buttons: primary, secondary, and plain.

Primary buttons are for primary actions, secondary buttons are for secondary actions. Plain buttons are styled to appear as links.

Standard link elements can also be styled to look like a button.

Button Examples

Button Accessibility

  • Buttons should always have a type attribute applied.
    • Buttons within forms automatically assume the type of submit.
    • Non-submit buttons should have the type of button.
